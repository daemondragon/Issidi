NetworkServer.Spawn et NetworkServer.destroy se font uniquement sur le serveur, donc les appeller par [Command]
(L'Intantiate du gameobject doit aussi se faire sur le serveur)

Script networké :
dériver la class de NetworkBehaviour au lieu de MonoBehaviour
tester la propriété hasAutority pour savoir qui est le proprio
enabled = hasAutority;// pour activer/ désactivé le script en fonction de qui est le propriétaire

Création des balles :
Ajouter un script Bullet et Interpolation
Ne JAMAIS oublier de désactiver dans le rigidbody la gravité (sinon une deuxième gravité s'aplique seulement sur l'axe y)

Création des armes :
Ajouter un scrip weapon a la racine du joueur
Ajouter un script weaponOrientation et colorModifier sur le modèles du joueur.
Ajouter la balles en prefab sur le script weapon (drag and drop)
NE PAS OUBLIER d'ajouter la balles que l'on veut faire spawner dans le NetworkManager -> Spawnable prefab (sinon ca spawn pas)

Ajout des personnages en reseaux:
Dans UI, ouvrir le CharacterFactory et ajouter les personnage dans la liste.
ATTENTION : l'ordre de la liste est le même que le weapon_index quand on veut créer un perso : Si l'on appelle avec 0 comme index, cela fera
spawn le premier élément de la liste.
NE PAS OUBLIER d'ajouter les personnage dans le NetworkManager, comme pour les balles.

Ajout de point de respawn :
Dans le MasterSpawner se trouve plusieurs type de spawn, chaque team pouvant avoir un nombre quasi infini de spawn, le choix au respawn
est aléatoire. Copier coller un spawn et le mettre avec les autres pour que cela marche.
Le MasterSpawner ne permet que de regrouper les spawns, il n'est pas obligatoire, mais merci de les laisser à l'interieur pour plus de claireté
dans la hiérachie des objets.
NE PAS OUBLIER de tagger les spawns (et pas le MasterSpawner pour qu'ils puissent être détectés)
